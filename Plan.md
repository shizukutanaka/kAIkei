# kAIkei — 統合バックオフィスプラットフォーム 開発計画書

> 本ドキュメントは、kAIkeiの全仕様（第1部〜第80部）を体系的に整理し、
> 開発チームが実装に着手できるようフェーズ・タスク・依存関係を定義したマスタープランである。

---

## 1. プロジェクト概要

| 項目 | 内容 |
|------|------|
| **製品名** | kAIkei |
| **形態** | マルチテナント SaaS型 統合バックオフィスプラットフォーム |
| **対象市場** | 日本の中堅〜大企業、官公庁 |
| **コアDB** | PostgreSQL（UUID主キー・履歴保持・Event Sourcing） |
| **アーキテクチャ** | API駆動・マイクロサービス・プラグイン拡張基盤 |
| **AI統合** | OCR・仕訳推論・税務予測・異常検知・プロセスマイニング |

### 4.2 設計原則

- **主キー**: 全テーブル UUID（`gen_random_uuid()`）
- **履歴管理**: Valid From-To / Event Sourcing パターン
- **金額型**: `NUMERIC(15, 4)` — 端数処理の精度担保
- **論理削除**: `is_deleted` / `is_voided` — 物理削除は原則禁止
- **タイムスタンプ**: 全テーブル `created_at` / `updated_at`（`TIMESTAMP WITH TIME ZONE`）
- **マルチテナント**: `tenant_id` + Row Level Security (RLS)
- **暗号化**: 口座番号等の機密情報はアプリケーション層で暗号化保管

## 6. AIエージェントアーキテクチャ

### 6.1 エージェント協調フロー

```
[証憑PDF追加]
    │
    ▼
[OCR Agent] ── テキスト・明細表・インボイス番号抽出
    │
    ▼
[Document Agent] ── 文書分類・メタデータ抽出・電帳法アーカイブ
    │
    ▼
[Journal Agent] ── 勘定科目・税区分・金額をLLMで推定
    │
    ▼
[Tax Agent] ── 税区分検証・インボイス番号検証・税額計算整合性
    │
    ▼
[Validation Agent] ── 貸借バランス・科目有効性・電帳法要件チェック
    │
    ▼
[Approval Agent] ── 信頼度スコア判定 → 自動ドラフト or 要確認タスク生成
```

### 6.2 AI説明責任（Explainability）要件

各AI推論結果は以下を必ず保持する：

| 項目 | 説明 |
|------|------|
| 推論結果 | 推奨勘定科目・税区分・金額 |
| 推論理由 | 「過去24件の同一取引先・同一摘要で95%が消耗品費」等 |
| 根拠データ | 参照した過去仕訳ID・ルールID |
| 適用ルール | 適用された業務ルール・法令ルールのID |
| AIモデル | モデル名・バージョン |
| 信頼度スコア | 0.00〜100.00 |
| 推論日時 | タイムスタンプ |
| 推論時間 | ミリ秒 |

### 6.3 AI自動コミット閾値

| 信頼度 | アクション | Feature Flag |
|--------|-----------|-------------|
| 90%以上 | 自動仕訳ドラフト生成（ワンクリック確認） | `ai_auto_draft` |
| 90%未満 | 「要確認」タスクをダッシュボードに表示 | — |
| 自動コミット | 信頼度推移が安定した企業のみON可能 | `ai_auto_commit`（個別企業フラグ） |

## 7. セキュリティ・コンプライアンス要件

### 7.1 認証・アクセス制御

| 要件 | 実装 |
|------|------|
| パスワード認証 | デフォルト（bcrypt ハッシュ） |
| SSO（SAML 2.0） | Microsoft Entra ID / Okta 連携 |
| SSO（OIDC） | Google Workspace / その他OIDCプロバイダー |
| MFA強制 | テナント単位でON/OFF切替 |
| IP帯域制限 | CIDR配列での許可リスト・ミドルウェアレベル遮断 |
| セッション管理 | JWT + Refresh Token / SSOセッション |

### 7.2 職務分掌（SoD）競合ルール

| 権限A | 権限B | 判定 |
|-------|-------|------|
| 支払登録 | 支払承認 | 禁止 |
| マスター変更 | 承認 | 禁止 |
| AI設定変更 | AI監査停止 | 禁止 |
| 銀行設定 | 振込実行 | 要承認 |

### 7.3 電子帳簿保存法（電帳法）対応

| 要件 | 実装 |
|------|------|
| 真実性の確保 | SHA-256ハッシュ・外部タイムスタンプ・改ざん検知バッチ |
| 可視性の確保 | 3軸検索（取引年月日・取引金額・取引先名称）の複数条件AND/OR検索 |
| 検索要件 | `idx_doc_search_requirements` 複合インデックス |
| 削除禁止 | `is_deleted` 論理削除のみ・物理削除は原則禁止 |

### 7.4 ディザスタリカバリ

| 項目 | 目標値 |
|------|--------|
| RPO（目標復旧時点） | 5分以内（WALアーカイブ最大タイムラグ） |
| RTO（目標復旧時間） | 3時間以内 |
| フルバックアップ | 毎日 02:00（別リージョンへエクスポート） |
| WAL継続アーカイブ | リアルタイム（トランザクション毎） |
| PITR | 任意時点のタイムスタンプ指定復元 |
