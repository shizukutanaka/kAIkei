# kAIkei — BPMN業務フロー仕様書

> 本ドキュメントはkAIkeiの日次・月次・年次業務プロセスをBPMN 2.0準拠のテキスト表現で定義する。
> 実装時はcamunda-modeler等でBPMN XML化し、ワークフローエンジン連携を可能とする。

---

## 1. 日次業務フロー

### 1.1 仕訳入力〜承認〜確定

```
[開始] → 仕訳起票 → バリデーションエンジン(VAL-001〜005)
  → [貸借一致?]
    ─No→ [エラー表示] → 修正 → バリデーションエンジン(再)
    ─Yes→ SoDチェック[起票者≠承認者?]
      ─No→ [エラー: 職務分掌違反] → 終了
      ─Yes→ 承認ワークフローへ送付
        → [承認者: 確認]
          ─差戻し→ [起票者: 修正] → バリデーション(再)
          ─承認→ 仕訳確定(posted) → 月次残高更新
            → Webhook発火(journal.posted)
            → 監査エンジン非同期実行
              → [リスク検知?]
                ─Yes→ audit_detection_logs登録 → [監査担当者: 確認]
                ─No→ 終了
```

### 1.2 AI仕訳推論フロー

```
[開始: 証憑アップロード] → OCRエンジン(Tesseract/Google Vision)
  → テキスト抽出 → AI推論エンジン(GPT/Claude)
    → 勘定科目マスタ照合 → 仕訳提案生成
      → ai_inference_logs登録(applied_status=pending_review)
        → [ユーザー: レビュー]
          ─適用→ 仕訳作成API呼出 → ai_inference_logs(applied_status=applied)
          ─修正→ 修正UI表示 → 仕訳作成API(modified_diff記録) → applied_status=modified
          ─却下→ applied_status=rejected → 終了
```

### 1.3 証憑アーカイブフロー

```
[開始: ファイル選択] → ファイルアップロード
  → SHA-256ハッシュ計算 → ハッシュ重複チェック
    ─重複あり→ [警告: 既存アーカイブ済み] → 終了
    ─重複なし→ メタデータ入力(取引日・金額・取引先名)
      → オブジェクトストレージ保存(S3互換)
        → タイムスタンプトークン取得(外部TSA)
          → archived_documents登録
            → Webhook発火(document.archived)
              → 終了
```

### 1.4 銀行明細取得〜自動消込

```
[開始: スケジューラ(毎朝6:00)] → 銀行API/スクレイピング実行
  → bank_statement_details一括登録
    → 未消込明細を取得
      → マッチングエンジン実行
        → [消込候補あり?]
          ─Yes→ 候補リスト表示 → [ユーザー: 確認]
            ─承認→ 消込仕訳自動生成 → is_reconciled=TRUE → 終了
            ─却下→ 手動消込画面へ → 終了
          ─No→ [手動消込待ち] → 終了
```

## 2. 月次業務フロー

### 2.1 月次決算クローズ

```
[開始: 月末] → 全仕訳の承認完了確認
  → [未承認仕訳あり?]
    ─Yes→ [承認者へリマインド通知] → 待機(3営業日)
      → [仍未承認?]
        ─Yes→ 上長へエスカレーション通知 → 強制確定 or 繰越
        ─No→ 次へ
    ─No→ 次へ
      → 月次残高再計算(monthly_balances更新)
        → 試算表(TB)自動生成
          → 帳簿一貫性検証(ledger-check実行)
            ─ドリフト検知→ [エラー: 帳簿不整合] → 修復バッチ実行 → 再検証
            ─OK→ 月次締めステータス更新(closed)
              → Webhook発火(monthly.closed)
                → 月次レポート自動生成(TB/損益/貸借対照表)
                  → 終了
```

### 2.2 給与計算フロー

```
[開始: 給与担当者] → 対象年月選択
  → 勤怠データ取り込み(外部API/CSV)
    → 社員マスタ・給与規定照合
      → 社会保険料計算(健康保険/厚生年金/雇用保険)
        → 源泉所得税計算(年末調整シミュレーション)
          → payroll_records一括生成
            → [dry_run?]
              ─Yes→ プレビュー表示 → [承認?]
                ─Yes→ 給与仕訳自動生成 → e-Gov送信(社会保険届出)
                  → 振込データ生成(全銀フォーマット)
                    → 給与明細発行(電子/紙)
                      → 終了
                ─No→ 修正 → 再計算
              ─No→ 本番実行 → 上記と同じフロー
```

### 2.3 固定資産減価償却フロー

```
[開始: 月初スケジューラ] → 全有効資産(assets where status=active)取得
  → [資産ごとに]
    → 償却方法判定(straight_line / declining_balance / double_declining)
      → 当月償却額計算
        → accumulated_depreciation更新
          → current_year_depreciation加算
            → 償却仕訳自動生成(journal_headers source_type=depreciation)
              → [全資産完了?]
                ─No→ 次の資産へ
                ─Yes→ サマリレポート生成 → 終了
```

### 2.4 消費税中間申告フロー

```
[開始: 期中(指定月)] → 対象期間の課税仕入・課税売上集計
  → インボイス区分別集計(qualified / transitional_80 / transitional_50)
    → 仕入控除額計算
      → [簡易課税?]
        ─Yes→ みなし仕入率適用 → 税額計算
        ─No→ 個別対応方式/一括比例配分方式 → 税額計算
          → 中間申告書データ生成
            → e-Tax送信
              → [送信成功?]
                ─Yes→ 申告完了ログ保存 → 終了
                ─No→ エラー表示 → [再送信 or 手動申告] → 終了
```

## 3. 年次業務フロー

### 3. 年末調整フロー

```
[開始: 12月] → 全社員の年間所得集計
  → 扶養控除・配偶者控除・社会保険料控除の確認
    → [マイナンバー提出済み?]
      ─No→ [提出催促通知] → 待機
      ─Yes→ 年末調整計算実行
        → 過不足税額算出
          → [還付?]
            ─Yes→ 還付金支給(給与に上乗せ) → 源泉徴収票作成
            ─No→ 追徴課税(給与から天引) → 源泉徴収票作成
              → e-Tax送信(源泉徴収票)
                → 社員へ電子交付 → 終了
```

### 3. 年次決算フロー

```
[開始: 期末] → 月次クローズ完了確認(全月)
  → 棚卸資産評価(原価法/低価法)
    → 固定資産の期末評価(除却・売却の処理)
      → 減価償却の最終調整
        → 引当金の見直し(貸倒引当金/賞与引当金等)
          → 税務調整(損金不算入項目の洗い出し)
            → 別表四・別表五(一)作成
              → 申告書データ生成
                → [AI税務リスク検知]
                  → [リスクあり?]
                    ─Yes→ [税理士/担当者: 確認] → 修正 → 再検査
                    ─No→ e-Tax送信(法人税/消費税/地方税)
                      → [送信成功?]
                        ─Yes→ 申告完了 → 確定申告書控え保存(電帳法)
                        ─No→ エラー処理 → 再送信
                          → 翌期首残高繰越 → 終了
```

### 3. データ移行フロー（他社会計ソフト→kAIkei）

```
[開始] → 移行元ソフト選択(弥生/freee/MF/Dreamvison)
  → CSV/APIエクスポート
    → フォーマット変換(移行元別アダプタ)
      → Dry-runシミュレーション
        → [エラー/警告あり?]
          ─Yes→ マッピング修正 → 差分レポート確認 → 再シミュレーション
          ─No→ [ユーザー: 本番移行承認]
            → バッチインポート実行
              → [進捗モニタ]
                → [完了?]
                  ─No→ 待機・再試行
                  ─Yes→ 移行結果レポート表示
                    → 残高検証(移行元TB vs kAIkei TB)
                      → [一致?]
                        ─Yes→ 移行完了 → ロールバックポイント削除 → 終了
                        ─No→ [差分分析] → [修正 or ロールバック] → 終了
```

## 4. 運用・保守フロー

### 4.1 バックアップ・リカバリ

```
[毎日 02:00] → フルバックアップ実行(別リージョンへエクスポート)
  → [成功?]
    ─Yes→ バックアップ完了ログ保存 → 終了
    ─No→ [アラート通知(運用担当)] → 再試行(最大3回)
      → [復旧?]
        ─Yes→ 終了
        ─No→ エスカレーション(管理者) → 手動復旧作業

[WALアーカイブ] → リアルタイム(トランザクション毎) → S3互換ストレージへ保存

[PITR復元要求時] → 復元対象タイムスタンプ指定
  → WALアーカイブから該当時点までリカバリ
    → [RTO 3時間以内?]
      ─Yes→ 復元完了 → 整合性チェック → サービス再開
      ─No→ エスカレーション → 緊急対応
```

### 4.2 リリース・アップグレード

```
[開始] → Feature Flag確認(段階的ロールアウト)
  → ステージング環境デプロイ
    → 自動テストスイート実行(単体/結合/回帰)
      → [全テスト成功?]
        ─No→ [修正] → 再デプロイ
        ─Yes→ カナリアリリース(5%トラフィック)
          → [障害/エラー増加?]
            ─Yes→ 自動ロールバック → [原因分析] → 終了
            ─No→ 段階的拡大(25% → 50% → 100%)
              → [全量完了?]
                ─Yes→ リリース完了ログ → Feature Flag全量ON → 終了
                ─No→ 監視継続 → 次段階へ
```

### 4.3 障害対応フロー

```
[開始: アラート検知] → 障害レベル判定(P1/P2/P3)
  → [P1(重大)?]
    ─Yes→ 即時エスカレーション(オンコール担当)
      → 影響範囲特定 → 緊急対応(ロールバック/切り替え/再起動)
        → 復旧確認 → インシデントレポート作成 → 根本原因分析(RCA)
          → 再発防止策実施 → 終了
    ─No→ [P2(中)?]
      ─Yes→ 営業時間内対応 → 修正パッチ作成 → 次回リリースまたはホットフィックス
      ─No→ P3(軽微) → バックログ登録 → 定期メンテで対応 → 終了
```

## 5. AIエージェント協調フロー

### 5.1 月次クローズ時のAI協調

```
[月次クローズ開始]
  → 仕訳AIエージェント: 未起票の証憑有無スキャン
    → [未処理証憑あり?]
      ─Yes→ AI仕訳推論実行 → ユーザーレビュー待ち
      ─No→ 次へ
  → 監査AIエージェント: 全仕訳のリスクスキャン
    → [異常検知?]
      ─Yes→ audit_detection_logs登録 → 監査担当者へ通知
      ─No→ 次へ
  → 税務AIエージェント: 当期着地利益・税額予測
    → 税務リスク警告生成(tax.risk_warning)
      → 次へ
  → 帳簿AIエージェント: 帳簿一貫性検証
    → [ドリフト?]
      ─Yes→ 自動修復 or 管理者通知
      ─No→ 次へ
  → 月次クローズ確定 → 終了
```

### 5.2 AI自動コミット判定フロー

```
[AI推論完了] → 信頼度スコア判定
  → [confidence >= 95 且つ 金額 < 100,000円 且つ 日常取引パターン一致?]
    ─Yes→ 自動コミット(仕訳確定)
      → 監査ログ記録(action=ai_approve)
        → Webhook発火(journal.posted)
          → 事後レビュー対象フラグ付与 → 終了
    ─No→ [confidence >= 70?]
      ─Yes→ ユーザーレビュー待ち(pending_review)
      ─No→ 低信頼度通知 + ユーザーレビュー待ち → 終了
```
