# kAIkei — テストカタログ

> 本ドキュメントはkAIkeiの全テスト階層・テストケース・カバレッジ目標を定義する。
> Plan.mdの品質保証目標（コアロジック90%以上・API自動テスト100%・重大障害ゼロ）を達成するための基準資料である。

---

## 1. テスト階層

| 階層 | 対象 | ツール | カバレッジ目標 |
|------|------|--------|--------------|
| 単体テスト | モデル・バリデーション・ユーティリティ | pytest / Vitest | 90%以上（コアロジック） |
| 結合テスト | API〜DB・サービス間連携 | pytest + httpx / Supertest | 主要パス100% |
| システムテスト | E2E・画面操作フロー | Playwright | 主要シナリオ100% |
| 回帰テスト | リリース毎の既存機能確認 | 自動化スイート | 全自動テスト再実行 |
| 性能テスト | 応答時間・スループット・大量データ | k6 / Locust | p95 < 500ms |
| セキュリティテスト | 認証・認可・注入・XSS | OWASP ZAP / Burp Suite | 重大脆弱性ゼロ |
| AI評価テスト | 推論精度・説明責任・ドリフト | カスタム評価スクリプト | 業務ごと目標値 |
| 法令適合テスト | 電帳法・インボイス・税計算 | カスタム検証スクリプト | 法令要件100%準拠 |

## 9. テスト自動化パイプライン

```
GitHub Push → GitHub Actions
  → [1] Lint (ruff/eslint)
  → [2] 単体テスト (pytest --cov / vitest --coverage)
  → [3] 結合テスト (pytest -m integration)
  → [4] システムテスト (Playwright)
  → [5] セキュリティスキャン (ZAP baseline)
  → [6] カバレッジレポート生成
  → [全成功?] → ステージング自動デプロイ
    ─No→ PRにコメント → 失敗
    ─Yes→ ステージング環境でE2E再実行 → 完了
```

## 10. テストデータ管理

| データ種別 | 作成方法 | 管理 |
|-----------|---------|------|
| マスタデータ | シードスクリプト（SQL） | リポジトリ内 `tests/fixtures/seed.sql` |
| 仕訳データ | ファクトリ（factory_boy / faker） | テスト実行時に動的生成 |
| 証憑ファイル | サンプルPDF/JPEG | `tests/fixtures/documents/` |
| AI推論テストデータ | アノテーション済みJSONL | `tests/fixtures/ai/` |
| 大量データ | バッチ生成スクリプト | `tests/scripts/bulk_generate.py` |
| 移行テストデータ | 弥生/freeeサンプルCSV | `tests/fixtures/import/` |
