# kAIkei — 法令ルールカタログ

> 本ドキュメントはkAIkeiが準拠すべき法令・規則をルールエンジン対応付けとして定義する。

## 7. ルールエンジン実装方針

### 7.1 ルールマスタ構成

```
legal_rule_library
├── tax_rules/          # 消費税・法人税・源泉徴収
├── social_insurance/   # 社会保険・雇用保険
├── e_book_law/         # 電帳法
├── labor_law/          # 労働基準法
└── rule_versioning/    # 法令改定時のバージョン管理
```

### 7.2 ルール適用フロー

```
[取引データ入力] → ルールカテゴリ判定
  → 該当ルールセット取得（有効期間フィルタ）
    → ルール条件評価（条件式エンジン）
      → [条件一致?]
        ─Yes→ アクション実行（税率適用・控除計算・警告等）
        ─No→ 次ルールへ
          → 全ルール評価完了 → 結果返却
```

### 7.3 法令改定への対応

| 改定パターン | 対応方法 | 例 |
|------------|---------|-----|
| 税率変更 | `tax_rules`テーブルの新レコード追加（valid_from更新） | 消費税率8%→10% |
| 限度額変更 | `tax_adjustment_rules`の`allowable_limit_formula`更新 | 交際費限度額変更 |
| 新制度創設 | 新ルールID追加・Feature Flagで有効化 | インボイス制度導入 |
| 経過措置 | 期間限定ルールとしてvalid_from/valid_to設定 | 80%控除の経過措置 |
| 廃止 | 該当ルールのvalid_toを設定 | 旧制度の廃止 |

### 7.4 ルールバージョン管理

```sql
CREATE TABLE legal_rule_versions (
    version_id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    rule_category VARCHAR(50) NOT NULL, -- 'tax', 'social_insurance', 'e_book_law', 'labor'
    effective_from DATE NOT NULL,
    effective_to DATE,
    version_number INT NOT NULL,
    change_summary TEXT NOT NULL,
    approved_by UUID NOT NULL,
    approved_at TIMESTAMP WITH TIME ZONE NOT NULL DEFAULT CURRENT_TIMESTAMP,
    CONSTRAINT uq_rule_version UNIQUE (rule_category, version_number)
);
```

> 法令改定時に新バージョンを作成し、過去バージョンとの差分を追跡可能とする。
> 過去年度の再計算時は該当年度のルールバージョンを適用する。
